# 数据库迁移安全

## 核心目标

- 在数据库变更上线前识别锁表、长事务、回填、索引与兼容性风险。

## 触发场景

- 新增表/改字段/加索引
- 数据回填
- 兼容迁移

## 默认检查项

1. 是否涉及大表 DDL
2. 是否存在无索引新查询路径
3. 是否需要回填与双写
4. 是否有先发库/先发代码顺序要求
5. 是否有回滚脚本

## 输出要求

- 输出迁移顺序
- 输出回滚顺序
- 对高风险 DDL 给出阻断
