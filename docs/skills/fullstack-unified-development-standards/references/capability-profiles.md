# 能力特性细则（合并版）

本文件保留原“严格对齐 / 统一新增 / 零漂移 / 实现层收敛 / 题库对齐”的关键能力点，统一由主技能调度。

## A. 严格对齐特性

### 1) 先冻结 10 项契约

1. 名称
2. 类型
3. 长度
4. 取值范围或枚举
5. 校验规则
6. 权限规则
7. 状态值
8. 状态流转规则
9. 文案文本
10. 注释与说明

### 2) 14 个影响面逐项核对

1. 数据库表字段
2. 后端实体
3. API 请求参数
4. API 响应参数
5. 前端页面展示
6. 前端表单
7. 前端表格
8. 前端状态
9. 前端校验规则
10. 权限控制
11. 用户可见文案
12. 图标与按钮
13. 状态流转规则
14. 文档、注释、测试

## B. 统一新增特性

新增需求默认按以下顺序推进：

1. 定义名字
2. 定义类型
3. 定义权限
4. 定义状态
5. 定义接口
6. 定义页面
7. 定义校验
8. 定义文档
9. 全端自检

## C. 零漂移特性

### 1) `global` 模式常见固定项

- 角色层级：`超级管理员 -> 管理员 -> 普通用户 -> 访客`
- 异常分类：`参数异常 -> 业务异常 -> 权限异常 -> 数据不存在 -> 重复提交 -> 系统异常`
- 目录建议：`controller/service/dao/entity/vo/dto/mapper/config/utils`
- 状态流模板（按项目既有契约择一，禁止混用）：
  - 新增型：`新增 -> 待审核 -> 已启用 -> 已禁用 -> 已删除`
  - 工作流型：`草稿 -> 待审核 -> 已通过 -> 已驳回 -> 已上线 -> 已下架`

### 2) 全链路一致要求

- 字段命名一致
- 类型一致
- 接口响应契约一致
- 公共组件、Hook、工具与请求封装一致
- 状态流与权限规则一致
- 校验、错误处理、日志一致
- 文档与测试断言一致
- 实现方式一致，禁止通过补丁式兼容、外围适配或双轨并存长期保留旧坏写法
- 已存在更优公共层或统一方案时，应直接替换旧实现并完成收敛

### 3) API 契约防漂移子守卫

- 主守卫链默认自动调用 `api-schema-drift-checker`。
- 重点覆盖：
  - OpenAPI/Swagger 与后端模型一致性
  - 后端契约与前端类型一致性
  - 前端请求方法/路径与 OpenAPI 一致性
- `final` 阶段出现高风险漂移应阻断交付并先修复。

### 4) 题库固定契约子守卫

- 主守卫链默认自动调用 `question-bank-contract-enforcer`。
- 重点覆盖：
  - 仅允许五模块：`user`、`userAuth`、`knowledge`、`question`、`task`
  - 接口响应包：`{ code, message, data }`
  - 契约外数据进入 `extJson`
- 单模块改动边界与 ID 关联规则
- `final` 阶段出现高风险漂移应阻断交付并先修复。

### 5) RBAC 权限对齐子守卫

- 主守卫链默认自动调用 `rbac-alignment-guard`。
- 重点覆盖：
  - 后端接口鉴权证据
  - 前端路由/菜单/按钮权限控制证据
  - 角色与权限键前后端一致性
- `final` 阶段出现高风险权限问题应阻断交付并先修复。

### 6) 状态流统一子守卫

- 主守卫链默认自动调用 `state-machine-alignment`。
- 重点覆盖：
  - 后端状态集合与前端状态集合一致性
  - 后端流转边与前端流转边一致性
  - 状态敏感按钮与流程入口的流转保护证据
- `final` 阶段出现高风险状态流问题应阻断交付并先修复。

### 7) 错误码统一治理子守卫

- 主守卫链默认自动调用 `error-code-governor`。
- 重点覆盖：
  - 后端异常改动是否具备错误码映射证据
  - 前后端错误码集合与命名空间一致性
  - 错误键命名一致性与同一码语义一致性
- `final` 阶段出现高风险错误码问题应阻断交付并先修复。

### 8) 全栈联动测试矩阵子守卫

- 主守卫链默认自动调用 `fullstack-test-matrix`。
- 重点覆盖：
  - 基于改动生成必测场景矩阵（正常/异常/边界/前后端/API/数据/权限/状态）
  - 识别“实现有改动但测试未更新”与关键场景漏测
  - 输出补测建议与矩阵化证据
- `final` 阶段出现高风险漏测应阻断交付并先修复。

### 9) 交付文档同步子守卫

- 主守卫链默认自动调用 `delivery-doc-sync`。
- 重点覆盖：
  - 实现改动与 README/接口/状态/权限/测试/发布说明的同步一致性
  - 识别“代码已改但文档未更新”与关键章节缺失
  - 输出文档同步矩阵与补文档建议
- `final` 阶段出现高风险文档漂移应阻断交付并先修复。

### 10) 实现层收敛子守卫

- 主守卫链默认自动调用 `component-reuse-shared-logic-guard`。
- 重点覆盖：
  - 重复实现是否已抽为公共组件、公共 Hook、公共函数
  - 页面请求是否统一走 `request`
  - 分页结构、时间字段、`extJson` 与基础实体字段是否统一
  - 公共层是否夹带业务写死逻辑，是否具备 `props`、事件、默认值、插槽等复用能力
- 不替代 API、题库契约、RBAC、状态机、错误码、测试、文档子守卫。
- `final` 阶段出现高风险实现漂移应阻断交付并先修复。

## D. 题库特性（`questionBank` 模式）

- 仅允许模块：`user`、`userAuth`、`knowledge`、`question`、`task`
- 仅允许修改用户指定模块
- 契约外业务字段统一进 `extJson`
- 表关系仅允许 ID 关联
- 响应包固定为 `{ code, message, data }`

固定契约见 [question-bank-fixed-contract.md](question-bank-fixed-contract.md)。
