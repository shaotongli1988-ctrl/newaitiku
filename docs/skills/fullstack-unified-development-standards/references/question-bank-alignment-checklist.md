# 题库对齐检查清单

每个主要改动批次后和最终交付前都必须执行本清单。

## 七项必检

1. 仅修改用户指定模块
2. 数据库字段与固定契约一致
3. 数据库字段、后端字段、接口字段一致
4. 接口字段、前端字段、页面展示一致
5. 契约外数据进入 `extJson`
6. 表关系遵循 ID 关联规则
7. 响应包符合 `{ code, message, data }`

## 修复规则

任一检查失败都要：

1. 立即修复实现
2. 重跑失败项
3. 再跑完整七项检查后再交付

## 模块级输出

`questionBank` 模式统一报告中，需按模块输出：

1. `user`
2. `userAuth`
3. `knowledge`
4. `question`
5. `task`

每个模块至少标注：

- 固定字段是否一致
- 响应包是否一致
- 契约外字段是否进入 `extJson`
- 关联是否遵循 ID 规则
- 页面与测试是否同步
