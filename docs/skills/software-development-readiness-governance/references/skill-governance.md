# 开发前准备总技能治理说明

## 一、目标

- 在进入开发前把“做什么、怎么验、页面长什么样、方案怎么定”一次性说清楚。

## 二、默认编排顺序

1. `requirements-freeze-guard`
2. `acceptance-criteria-builder`
3. `prd-ui-traceability-guard`
4. `architecture-decision-recorder`

## 三、交接关系

- 本技能完成后，开发实施默认交给 `fullstack-unified-development-standards`。
- 开发完成后，终检与上线门禁默认交给 `software-delivery-unified-governance`。

## 四、阻断规则

- `start` 只提示风险，不阻断。
- `batch/final` 出现 `high` 风险时，原则上不允许进入开发。
