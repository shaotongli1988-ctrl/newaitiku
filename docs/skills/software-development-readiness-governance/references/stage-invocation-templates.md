# 三阶段总技能调用模板

## 一、固定映射

| 阶段 | 默认总技能 | 你可以直接说的话 |
| --- | --- | --- |
| 开发前 | `software-development-readiness-governance` | `用开发前总技能，先把需求、验收、PRD/UI、技术方案一次性定义清楚。` |
| 开发中 | `fullstack-unified-development-standards` | `用开发中总技能开始做，边开发边对齐，别跑偏。` |
| 开发后 | `software-delivery-unified-governance` | `用开发后总技能检查一切，给我提测/上线前结论。` |

统一万能入口 skill：`three-stage-skill-orchestrator`

## 二、最稳的短句

- 开发前：`开发前定义好一切`
- 开发中：`开发中别跑偏`
- 开发后：`开发后检查一切`
- 万能入口：`三阶段总技能`

## 三、展开版模板

### 开发前

`用开发前总技能处理这个需求：先冻结范围、验收标准、PRD/UI 追踪关系、技术方案和开发边界，再告诉我是否允许进入开发。`

### 开发中

`用开发中总技能处理这个需求：按全端统一方式实施，持续对齐数据库、后端、接口、前端、权限、状态、错误、文档和测试，别让我开发跑偏。`

### 开发后

`用开发后总技能处理这个版本：先做全端终检，再做提测/上线前检查，输出阻断项、必须补齐项和最终结论。`

### 万能调用

`用三阶段总技能处理这个需求：先判断现在属于哪个阶段，再进入对应总技能。`

## 四、切换口诀

- 还没开始写实现：用开发前总技能。
- 已经开始写代码和联调：用开发中总技能。
- 代码基本完成，要提测或上线：用开发后总技能。

## 五、老板口语版

- 开发前：`这个先别急着做，先把需求、验收、页面和方案都定清楚。`
- 开发中：`现在开始做，但过程里别跑偏，前后端和页面都要一直对齐。`
- 开发后：`这版做完后先全面检查一遍，再告诉我能不能提测、能不能上线。`

## 六、工程口语版

- 开发前：`先走开发前总技能，把范围、验收、PRD/UI 追踪和技术方案冻结掉。`
- 开发中：`进入开发中总技能，按全端统一方式实施，边做边对齐。`
- 开发后：`切开发后总技能，先跑全端终检，再看提测和上线门禁。`

## 七、超短指令版

- 开发前：`先定义`
- 开发中：`开始做，别跑偏`
- 开发后：`做完了，全面检查`

## 八、按钮口令版

- 推荐主口令：`三阶段总技能`
- 语义：交给 `three-stage-skill-orchestrator` 做阶段路由，再进入对应总技能。
- 使用场景：你想一句话触发整套流程时，直接说 `三阶段总技能处理这个需求` 即可。
