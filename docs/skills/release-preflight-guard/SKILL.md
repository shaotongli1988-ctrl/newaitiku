# 发布前检查

## 核心目标

- 在发布前阻断缺配置、缺备份、缺监控、缺回滚的上线风险。

## 触发场景

- 灰度前检查
- 正式上线前检查
- 版本发布评审

## 默认检查项

1. 构建产物是否存在且版本一致
2. 环境变量与配置项是否齐全
3. 数据库脚本与应用版本是否匹配
4. 监控、日志、告警是否已开启
5. 是否具备回滚与备份条件

## 输出要求

- 输出 high/medium/low 风险清单
- 明确是否允许上线
- 对每个阻断项给出修复动作
