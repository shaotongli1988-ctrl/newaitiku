# 回滚就绪性检查

## 核心目标

- 确保应用、配置、数据库三层都可回滚。

## 触发场景

- 发布前评审
- 灰度前确认
- 重大版本上线

## 默认检查项

1. 应用版本是否可回退
2. 数据库变更是否可逆或兼容降级
3. 配置变更是否可回滚
4. 功能开关是否可止血
5. 是否有明确回滚步骤与负责人

## 输出要求

- 输出回滚步骤
- 标识不可逆变更
- 对无回滚能力项给出阻断
