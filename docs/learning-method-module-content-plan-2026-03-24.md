# 学习方法模块内容冻结（V1）

## 1. 模块目标

- 在“我的题库”下新增“学习方法”模块，提供可执行、可练习、可复盘的方法论内容，不做泛励志内容。
- 首批上线 12 个高频有效方法，覆盖「理解、记忆、练习、复盘、时间管理」五类学习场景。
- 每个方法都要和题库动作打通，确保用户“看完就能练”。

## 3. 方法卡片统一模板（前后端同构）

- `methodCode`：唯一编码（如 `M01`）。
- `methodName`：方法名。
- `oneLineIntro`：一句话解释（<= 40 字）。
- `useWhen`：适用场景（2-4 条）。
- `steps`：执行步骤（3-5 步）。
- `commonMistakes`：常见误区（2-4 条）。
- `questionBankActions`：题库联动动作（1-3 条）。
- `starterTask`：5-10 分钟入门任务（1 条）。
- `difficultyLevel`：`L1/L2/L3`。
- `estimatedMinutes`：建议时长（分钟）。

## 4. 首版验收标准（Given / When / Then）

- Given 用户进入“我的题库-学习方法”，When 打开任意方法卡，Then 可看到完整的「适用场景、步骤、误区、题库动作、入门任务」。
- Given 用户点击“开始练习”，When 系统跳转题库训练，Then 必须自动带入该方法对应的练习策略（如混编、限时、错题二刷）。
- Given 用户完成一次方法练习，When 返回方法页，Then 可看到本次练习记录（完成时间、正确率、复盘结果）。
- Given 管理端新增方法卡内容，When 发布后，Then 学生端可按方法编码稳定读取，不影响已有题库练习链路。

## 5. 内容边界（避免跑偏）

- 不收录纯鸡汤、无步骤、无练习动作的“方法”。
- 不收录与题库行为无法关联的泛心理学内容。
- 首版不做复杂测评问卷，仅保留“方法介绍 + 一键练习 + 练后复盘”闭环。

## 6. 建议排期（两周节奏）

- 第 1 周：冻结 12 个方法文案与卡片字段，完成管理端录入与学生端展示。
- 第 2 周：打通题库联动动作与练后记录，完成首轮可用性验收。
