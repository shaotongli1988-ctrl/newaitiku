# 学生自由练习筛选重构说明（2026-03-21）

## 影响页面

- 页面：`/student/practice`
- 文件：`frontend/src/views/Student/Practice.vue`

## 前端说明

本次重构把原先“科目下拉 + 通用知识树 cascader”的深层筛选方式，改成了更适合高密度知识定位的 Practice 工作台：

- 顶部筛选栏拆为 5 段：
  - 科目标签
  - 章节标签
  - 学科门类标签
  - 联考组标签
  - L5 可搜索级联选择器
- 中部新增科目卡片：
  - 卡片标题展示当前科目
  - 卡片内容使用胶囊式章节标签展示当前科目下的 L4 章节
  - 章节标签展示题量、章节状态，并对未通过章节守卫的项目做禁用处理
- 下部答题区改为“双栏工作台”：
  - 左侧为当前筛选结果下的题目总览列表
  - 右侧为当前激活题目的作答面板与 AI 批改进度

## 状态流说明

本次页面保留了学生练习会话、答案自动保存和 AI 批改链路，新增与调整的主要状态约束如下：

- Query 范围字段固定使用：
  - `examCategoryCode`
  - `jointExamGroupCode`
  - `subjectCode`
  - `chapterCode`
  - `chapterName`
  - `pointCode`
  - `pointName`
  - `knowledgeId`
  - `pathLabel`
- 点击上层筛选项时，必须清空其下游路径字段，避免出现“旧 L5 仍挂在新科目下”的脏状态。
- 手动切换筛选范围时，会同步清理弱项强化与风险修复模式下的临时路由字段，确保自由练习以用户当前手选范围为准。
- L5 精准定位仍通过知识树 `L4 -> L5` 路径映射回 `knowledgeId / pointCode`，不会改动既有练习题接口契约。

## 测试说明

已完成的静态验证：

- `cd frontend && npm run build`

建议补充的页面级回归：

1. 首次进入 `/student/practice`，确认页面会按学生当前作用域补齐默认 `examCategoryCode / jointExamGroupCode / subjectCode`。
2. 切换科目后，章节标签与 L5 搜索候选同步刷新，题目列表联动更新。
3. 点击任一 L4 章节胶囊，确认 query 中仅保留章节级路径，不残留旧的 `knowledgeId / pointCode`。
4. 在 L5 级联中搜索并选中具体原子考点，确认下方题目总览只显示该考点下题目。
5. 在题目总览中切换题目、作答并提交，确认答案自动保存、AI 批改进度卡正常刷新。
6. 从弱项强化或风险修复链路进入后，不操作筛选时原有聚焦行为保持；手动改筛选后应切换为当前自由练习范围。
