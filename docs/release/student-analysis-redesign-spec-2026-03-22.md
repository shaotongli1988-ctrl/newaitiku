# 知识诊断页面重构说明（2026-03-22）

## 1. 阶段与结论

- 三阶段路由结果：`开发前（readiness）`
- 推荐技能：`software-development-readiness-governance`
- 当前结论：允许继续细化方案，但进入实现前需要补齐页面结构、验收口径与颜色映射规则。

本次重构目标不是再做一个“更花哨的图”，而是把现有知识诊断页从“圆形全景探索”调整为“更容易读懂、更容易行动”的结构，同时保持学生端现有蓝灰基调与统一 token 不变。

## 2. 当前基调冻结

- 主色：`--qb-primary-student` / `--zsb-color-primary` = `#2f54eb`
- 主色浅底：`--qb-primary-soft-bg` / `--zsb-color-primary-light` = `#f0f5ff`
- 页面底色：`--qb-bg-body` = `#f0f2f5`
- 卡片底色：`--qb-bg-card` = `#ffffff`
- 标题文字：`--qb-text-heading` = `#0f172a`
- 正文文字：`--qb-text-secondary` = `#595959`
- 成功态：`--qb-success` = `#52c41a`
- 警示态：`--qb-warning` = `#faad14`
- 风险态：`--qb-danger` = `#f5222d`

### 2.2 颜色使用原则

- 不更换品牌主色，仍以“极客蓝 + 浅灰底 + 白卡片”为主框架。
- 不做高饱和拼色面板，不把每个模块染成独立主题色。
- 蓝色只承担“结构主线、核心数据、主按钮、重点引导”。
- 红黄绿只承担“诊断状态”，不承担页面主视觉。
- 大面积背景继续使用浅蓝白渐变和浅灰底，避免页面突然变重或变花。

## 3. 页面方向

视觉方向：沿用当前学生端的“专业、清晰、带轻微科技感”的蓝灰卡片体系，把原来的圆形主画布改为“总览卡 + 排行条 + 优先任务卡”。

页面结构改为两段式：

1. 第一屏：`诊断总览`
2. 第二屏：`优先攻克`

## 4. 页面结构

### 4.1 第一屏：诊断总览

目标：用户进入后 3 秒内看懂“整体水平如何、最弱的是哪里”。

保留现有 hero 区的语气，但改掉“全景探索”的阅读负担。

建议结构：

- 页头标题：`知识诊断`
- 主标题：`知识结构与薄弱点诊断`
- 辅助文案：`先看整体掌握情况，再按优先级补最值得解决的薄弱点。`
- 右侧总览卡：
  - 当前科目
  - 当前掌握度
  - 已练题量 / 总题量
  - 薄弱点数量

页头下方改为两块：

- 左侧：模块总览卡组
  - `数与代数`
  - `几何`
  - `函数`
  - `统计`
- 右侧：薄弱点排行榜

每个模块卡只保留：

- 模块名
- 掌握率
- 薄弱点数
- 状态标签：`基础稳定 / 需巩固 / 需优先补强`

薄弱点排行榜采用横向条形图，按弱到强排序，默认展示前 5 项。

### 4.2 第二屏：优先攻克

目标：用户看完结果后，不需要自己判断“先练什么”。

建议结构：

- 标题：`现在先学这 3 项，提升最快`
- 三张任务卡横排或竖排
- 每张卡包含：
  - 知识点名称
  - 当前掌握率
  - 推荐原因
  - 建议练习题数
  - 主按钮：`立即巩固`

推荐原因只用现有产品可理解的话术：

- `失分集中`
- `考试高频`
- `短期提分快`

避免重新引入复杂图谱说明，不让用户在第二屏继续“读图”。

## 5. 颜色映射

### 5.1 总览层

- 页面背景：继续使用 `--qb-bg-body`
- Hero 与总览区域背景：继续使用浅蓝白渐变，主色来自 `--qb-primary-soft-bg`
- 主标题与关键数字：使用 `--qb-text-heading`
- 主操作按钮与关键百分比：使用 `--qb-primary-student`
- 模块卡边框：使用 `--qb-primary-soft-border` 或 `--qb-border-muted`

### 5.2 诊断状态层

状态色只用在标签、条形图前景、状态点，不大面积铺底：

- `基础稳定`：`--qb-success`
- `需巩固`：`--qb-warning`
- `需优先补强`：`--qb-danger`
- `未形成画像`：使用 `--qb-text-subtle-8` + `--qb-bg-muted`

### 5.3 排行与任务层

- 条形图轨道底色：`--qb-bg-muted`
- 条形图前景默认从 `--qb-primary-student` 到其浅色渐变
- 如果条目为高风险，可在条尾或状态点使用 `--qb-danger`
- 任务卡背景保持白色或极浅蓝白底，不使用整块红底或黄底

### 5.4 禁止项

- 不新增紫色、荧光色、霓虹渐变
- 不使用大面积红色背景承载“薄弱项”
- 不再沿用多圈、多色块的圆形主视觉作为页面中心

## 6. 文案替换建议

当前文案里偏“看图”的表述应弱化，改成偏“诊断 + 建议”。

建议替换：

- `全景探索` -> `诊断总览`
- `局部探索` -> `章节诊断`
- `实时洞察` -> `当前诊断结论`
- `颜色负责分布，侧栏负责决策` -> `先看结果，再按优先级行动`

推荐标题组合：

- 页面标题：`知识诊断`
- 第一屏标题：`诊断总览`
- 第二屏标题：`优先攻克`

## 7. 交互规则

- 顶部总览卡点击模块后，薄弱点排行榜同步过滤到当前模块。
- 点击排行榜某一项，第二屏对应任务卡高亮。
- 点击任务卡主按钮，直接进入专项练习，不再经过二次说明弹层。
- 当用户暂无数据时，不展示空图谱，直接展示空状态卡和主动作按钮。

## 8. 移动端规则

- 第一屏改为纵向堆叠：总览卡在前，模块卡横滑，排行榜纵向排列。
- 第二屏任务卡改为单列卡片流。
- 移动端首屏只保留一个主要结论和一个主要动作，避免信息过满。

## 9. 验收标准

### 9.1 结果可读性

- Given 用户进入知识诊断页
- When 页面加载完成
- Then 用户首屏应能直接看到总掌握度、模块强弱摘要和薄弱点排行，而不是先阅读圆形图说明

### 9.2 行动可达性

- Given 页面已生成薄弱点诊断结果
- When 用户浏览到第二屏
- Then 页面应直接给出 3 个优先攻克项，每项均有明确行动按钮

### 9.3 视觉一致性

- Given 页面完成改版
- When 与首页、任务中心、题库页并排查看
- Then 知识诊断页应保持统一学生端蓝灰基调，不应出现独立的新主色体系

### 9.4 状态语义

- Given 页面出现绿色、黄色、红色状态表达
- When 用户查看模块卡、排行条或状态标签
- Then 颜色只表达诊断状态，不应承担页面主视觉，且语义保持一致
