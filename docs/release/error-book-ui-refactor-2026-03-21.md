# 错题本医疗诊断化重构说明（2026-03-21）

> 迁移说明：
> 本文记录的是 2026-03-21 当时的独立“错题本”页面重构。
> 当前学生端入口已统一迁移到 `我的题库 > 错题中心`，旧的 `/student/wrong-book` 仅保留重定向兼容。

## 影响页面

- 页面：`/student/wrong-book`
- 文件：`templates/wrong-book.html`
- 文件：`static/wrong-book.js`
- 文件：`static/style.css`

## 前端说明

本次重构将错题本从“纵向堆叠列表 + 右侧 JSON 明细”调整为更偏医疗诊断工作台的结构，重点做了以下改动：

- 顶部统计区改为 3 个并排的小圆环进度条，分别呈现覆盖率、近 7 天新增趋势和当前重灾章节占比。
- 主体布局改为 `180px` 左侧科目导航 + 右侧诊断卡片区，科目切换时会联动刷新当前错题列表与统计信息。
- 错题卡片内部改为双栏结构：
  - 左侧约 65%：仅保留一个微型 L5 考点标签，主区域优先展示题干和选项。
  - 右侧约 35%：使用灰底诊断侧栏集中展示 AI 诊断建议、L4 章节定位、错因、状态、掌握度与累计错题信息。
- 原先默认展开的答案/解析被收敛进卡片底部详情区，默认收起，点击“展开详情”后通过动画滑出。
- 原有“按错因生成专项卷”和“生成专属同类卷”能力保留，但调整为更紧凑的提示条和胶囊按钮样式，避免抢占题干视觉焦点。

## 状态流说明

本次重构未修改错题本核心接口契约，仍沿用以下链路：

- 科目导航通过 `/api/question-bank/student/wrong-book/summary` 的 `selectedSubjectCode / subjectRows / currentQuestionInsights` 驱动。
- 当前列表通过 `/api/question-bank/student/wrong-book/questions?page=1&size=100&subjectCode=...` 拉取。
- “完成复习”仍调用 `/api/question-bank/student/wrong-book/questions/{questionId}/review`，刷新当前科目数据。
- “生成专属同类卷”与“按错因生成专项卷”接口不变，仅更新了前端结果展示方式。
- 详情展开状态仅保存在前端内存中，不回写服务端，不影响既有学生记录结构。

## 测试说明

已完成的静态检查：

- `node --check static/wrong-book.js`

建议补充的页面级回归：

1. 进入 `/student/wrong-book` 后，确认默认选中 summary 返回的当前科目，且左侧科目导航宽度保持 180px。
2. 切换不同科目时，顶部 3 个圆环统计、右侧卡片列表和标题副文案同步更新。
3. 检查任意错题卡片：左侧优先展示题干和选项，右侧灰底区域显示 AI 诊断建议与 L4 定位。
4. 点击“展开详情”后，答案与解析应平滑滑出；再次点击后应收起。
5. 点击“完成复习”后，页面应刷新当前科目数据并提示复习次数。
6. 点击“生成专属同类卷”或任一错因专项卷按钮后，顶部结果提示条应展示最新试卷信息。
