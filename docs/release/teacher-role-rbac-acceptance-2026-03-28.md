# 老师双岗权限与超管开通同步验收（2026-03-28）

## 验收范围

- 业务范围：开题库权限开通，覆盖招生老师、教学老师、双岗老师。
- 角色范围：`super_admin`、`teacher`（`postTags` 支持 `recruit`、`teach`、`recruit+teach`）。
- 页面范围：教师首页、超管控制台、权限显隐与数据范围收敛。

## Given / When / Then

1. Given 超管在控制台为老师配置 `postTags=["recruit","teach"]`
   When 老师登录题库并进入首页
   Then 同时可见招生运营信息和教学工作信息，且菜单权限与接口权限一致。

2. Given 超管仅配置 `postTags=["recruit"]`
   When 招生老师进入学生目录与学情看板
   Then 仅展示该老师 `managedStudentIds` 范围内学生刷题信息，不得越权读取其他学生数据。

3. Given 超管将老师岗位从 `recruit` 切换为 `teach`
   When 老师刷新页面并重新请求数据
   Then 前端按钮显隐、路由访问、后端鉴权返回保持一致，不出现“前端可点后端403”的漂移。

4. Given 招生老师也需要查看学生刷题信息
   When 查询学生列表与刷题统计
   Then 可读取只读统计字段，不新增后端写接口，不修改接口结构。

## UAT 场景矩阵

- 正常路径：超管开通双岗 -> 老师登录 -> 首页模块与菜单均正确展示。
- 权限路径：单岗老师访问非授权模块 -> 返回 401/403 且前端不可见入口。
- 边界路径：`managedStudentIds` 为空 -> 返回空列表与空态文案。
- 异常路径：鉴权失败或 token 过期 -> 引导重新登录。

## 预期结果

- 验收口径统一：Given / When / Then 与接口行为一致。
- UAT 可追踪：每条场景都有预期结果和可复现入口。
