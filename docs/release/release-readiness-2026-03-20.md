## 二、本次范围

- 后端：`app/`
- 前端：`frontend/`（单入口 Vue + Vite SPA）
- 数据库：`data/schema.sql`

## 三、上线前必须满足

1. 数据库初始化脚本不得包含破坏性 `DROP TABLE`。
2. 前端源码与构建产物中不得暴露超管账号密码。
3. 后端核心查询避免使用 `SELECT *`。
4. 发布必须在非 `main/master` 的工作分支上完成。
5. 发布前需确认配置、监控、回滚条件。

## 四、验收标准

### 1. 登录与权限

- 超管、教师、学生均可通过 `/login` 正常登录。
- 未登录访问受保护页面会被重定向到 `/login`。
- 超管可访问管理后台，教师可访问教师端，学生可访问学生端。

### 2. 题库与组卷

- 题库列表可正常加载、搜索、分页。
- 题目详情、编辑、状态流转可正常执行。
- 教师端组卷入口可打开手动选题与 AI 组卷弹层。

### 3. 数据库安全

- 初始化脚本只创建缺失表与索引，不直接销毁已有数据。
- 现有数据库可在不删表前提下正常启动。

## 五、UAT 交接

### 环境入口

- 登录页：`/login`
- 超管首页：`/admin/home`
- 教师首页：`/teacher/home`
- 学生首页：`/student/home`

### 验收重点

1. 登录跳转是否正确
2. 教师端题库管理与组卷流程是否可用
3. 学生端首页、练习、我的题库子页是否可访问
4. 权限越权访问是否被拦截

## 七、回滚说明

1. 若前端静态资源发布异常，回滚到上一版静态资源。
2. 若后端代码发布异常，回滚到上一版应用包。
3. `data/schema.sql` 不再执行破坏性删表；若迁移异常，禁止直接重建生产库。
