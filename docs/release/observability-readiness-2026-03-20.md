# 可观测性说明（2026-03-20）

## 文档信息

- 文档类型：上线后监控与排查说明
- 适用版本：2026-03-20 发布批次
- 适用阶段：发布后 0-30 分钟重点观察
- 责任角色：研发、运维、测试

## 一、目标

- 为本次版本提供上线后的问题发现与排查依据。

## 二、当前已覆盖

- 前端请求统一走 `request`
- 后端关键服务已具备基础调用链
- 可通过总守卫继续识别“日志 / trace / metrics 证据不足”的模块

## 三、上线后重点关注链路

1. 登录链路
2. 题库列表查询链路
3. 教师组卷链路
4. 学生练习与任务链路

## 四、发布后观察项

### 1. 登录与鉴权

- 是否出现大量 401 / 403
- 登录是否存在异常跳转

### 2. 题库与组卷

- 题库查询是否慢
- 组卷接口是否超时
- AI 任务是否堆积

### 3. 学生端

- 首页是否白屏
- 练习提交是否失败
- 我的题库子页是否返回异常

## 五、日志与监控建议

1. 记录关键接口耗时
2. 记录异常堆栈与请求上下文
3. 记录登录、题库、组卷、练习关键路径的错误数
4. 发布后重点观察前 30 分钟日志与异常

## 六、关键链路观察矩阵

| 链路 | 关键接口 / 页面 | 重点观察 | 异常判定 |
| --- | --- | --- | --- |
| 登录与鉴权 | `POST /api/question-bank/auth/login/password`、`/login` | 登录成功率、401/403、异常跳转 | 连续失败、角色跳转错误、401/403 激增 |
| 学生首页 | `GET /api/question-bank/student/dashboard`、`/student/home` | 首页卡片、章节树、任务列表是否返回 | 200 但数据为空、白屏、接口超时 |
| 学生练习 | `GET /api/question-bank/student/practice/questions`、`POST /api/question-bank/student/practice/questions/{id}/submit` | 题目列表、提交成功率、耗时 | 题目空列表、提交失败、5xx 增多 |
| 教师组卷 | `GET /api/question-bank/papers/overview`、`POST /api/question-bank/papers/manual`、`POST /api/question-bank/papers/ai-generate` | 试卷列表、手动组卷、AI 组卷任务 | 组卷超时、任务堆积、列表报错 |
| 教师学情 | `GET /api/question-bank/analytics/summary`、`GET /api/question-bank/analytics/records`、`/teacher/analytics` | 统计摘要、学情记录、导出可用性 | 统计为空、分页异常、导出失败 |
| 消息中心 | `POST /api/question-bank/messages/send`、`GET /api/question-bank/messages?readStatus=unread`、`/messages` | 发送、未读、已读回执 | 已发送但未读列表不变、消息页面异常 |
