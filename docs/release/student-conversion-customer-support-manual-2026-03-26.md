# 学生转化闭环 MVP 客服手册

- 版本：v1.0（冻结）
- 日期：2026-03-26
- 适用角色：一线客服、值班客服、客服主管
- 关联功能：快诊、兑换码、订阅、模拟支付、首页权益态

## 1. 处理原则

- 所有判断以服务端接口返回为准，不以前端截图单独定责。
- 先确认账号身份与入口（学生入口/教师入口），再判断功能异常。
- 优先排查可自助恢复问题，避免无效升级。

## 2. 高频问题与标准回复

### 2.1 兑换码失败

- 用户话术：兑换码无效/不能用。
- 标准检查：
  - 是否为空码或格式错误。
  - 是否已被使用（一码一次）。
  - 当前账号是否已兑换过（一号一次）。
  - 是否过期。
- 建议回复：
  - 请确认兑换码输入无空格；若提示已使用/已过期，请联系发码方重新申请可用码。

### 2.2 支付后权益未生效（MVP 为模拟支付）

- 用户话术：点击开通后仍显示未开通。
- 标准检查：
  - `GET /student/subscription/status` 是否 `subscriptionActive=true`。
  - 是否在支付确认后立即刷新页面（可能存在短暂延迟）。
- 建议回复：
  - 请返回首页刷新后重试，如仍未生效请提供账号与操作时间。

### 2.3 登录后跳转异常

- 用户话术：一直跳页面/进不去首页。
- 标准检查：
  - 账号角色是否学生。
  - `dashboard.onboarding.completed` 是否为 `false`（未完成会跳到快诊）。
- 建议回复：
  - 当前账号需先完成首登快诊引导，完成后会自动恢复首页路径。

## 4. 升级策略

### 4.1 升级到研发（P1）

- 满足任一条件：
  - 同账号多次重试仍失败。
  - `subscription/status` 与页面展示矛盾。
  - 疑似分流死循环。

### 4.2 紧急升级（P0）

- 满足任一条件：
  - 大量学生无法完成快诊或无法开通。
  - 首页分流全面异常，主学习链路被阻断。

## 5. 值班检查清单

- 每日检查 3 次：
  - 兑换失败工单量是否突增。
  - 开通后未生效工单是否连续增长。
  - 路由跳转异常是否集中在某版本。

## 6. 版本冻结声明

- 本手册用于学生转化闭环 MVP 上线期。
- 如真实支付接入或订阅规则变更，必须在新版本手册更新后再对外执行。
