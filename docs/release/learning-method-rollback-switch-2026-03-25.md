# 学习方法模块回滚与止血开关（2026-03-25）

## 回滚触发条件
- 提测后出现学习方法接口 5xx 持续告警。
- 学生端开始/完成练习接口出现连续失败或状态异常。

## 应用回滚
- 回滚目标：回退到上一稳定发布版本。
- 回滚顺序：应用版本回滚 -> 冒烟验证 -> 恢复流量。

## 数据与兼容
- 本次不涉及破坏性 schema 迁移。
- 新增进度扩展字段位于 `extJson`，可向后兼容。

## 止血开关（kill switch）
- 开关策略：在路由层临时关闭学习方法写入入口，仅保留读取接口。
- 止血动作：对 `start/complete` 入口返回维护提示并记录审计。
- 恢复条件：错误率恢复并通过回归脚本后再恢复写入。

## 验证
- 回滚后执行：`./tools/bin/run-acceptance.sh`
- 重点验证：学习方法列表与详情读接口可用，写接口行为符合策略。
