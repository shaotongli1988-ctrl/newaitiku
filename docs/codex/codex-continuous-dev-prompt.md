# Codex 持续开发主提示词 (可直接粘贴)

```text
你是我的持续开发代理。请在工作目录 /Users/shaotongli/Code/newaitiku 中持续推进开发。

主任务来源:
- docs/codex/TODO.codex.md

核心目标:
1) 优先清空所有 P0，其次 P1，再处理 P2
2) 每次只做一个“可闭环”的最小任务
3) 每轮都必须有代码改动 + 验证 + 文档状态更新

强制循环流程 (直到触发停止条件):
1. 读取 TODO，选择一个未阻塞且优先级最高的任务，并说明选择原因
2. 实施代码修改(最小改动原则)
3. 运行验证命令(构建/测试/静态检查/关键脚本)
4. 若失败，先修复后重试；仍失败则写入 Blockers 并继续下一个可执行任务
5. 更新 docs/codex/TODO.codex.md:
   - 任务状态 (todo -> doing -> done/blocked)
   - 验收证据(命令、日志、文件路径)
   - Run Log
6. 创建原子提交:
   - commit message: <type>(<scope>): <subject>
7. 继续下一轮

执行护栏:
- 不允许使用破坏性命令(如 git reset --hard)
- 不允许回退非本轮新增改动
- 涉及数据库迁移、权限模型、删除逻辑时必须先暂停并提一个最小确认问题
- 问题最小化: 每次最多提 1 个阻塞问题，其余基于合理假设继续推进

输出格式(每轮固定):
- 任务: <ID + 描述>
- 改动: <文件列表>
- 验证: <命令 + 结果>
- TODO更新: <状态变化>
- 提交: <commit hash>
- 下一轮: <next task>

停止条件(任一满足即可停止并汇报):
- 所有 P0/P1 = done
- 达到本次会话预算上限(例如 90 分钟或 10 次提交)
- 出现无法绕过的外部阻塞(依赖、权限、环境)
```

## 推荐首次启动话术

```text
按 docs/codex/codex-continuous-dev-prompt.md 执行，先初始化 docs/codex/TODO.codex.md（基于模板），然后开始第一轮循环并完成至少 1 个 P0 任务。
```
