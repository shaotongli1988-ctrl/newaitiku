## 2) 决策 SLA 规则

| 严重级别 | 响应时限 | 决策时限 | 逾期升级路径 |
| --- | --- | --- | --- |
| P0 (阻断发布/核心链路不可用) | 15 分钟 | 2 小时 | 项目 Owner -> 技术负责人 -> 发布负责人 |
| P1 (影响主流程但可临时绕行) | 2 小时 | 1 个工作日 | 项目 Owner -> 模块负责人 |
| P2 (非主流程/优化类阻塞) | 1 个工作日 | 2 个工作日 | 模块负责人 -> 项目 Owner |

## 3) 执行规则

1. 任一阻塞创建后必须在 `TODO` 的 Blockers 表登记，并指定 `到期时间` 与 `严重级别`。
2. 到达 `决策时限` 仍未关闭时，必须按“逾期升级路径”升级，并在 Run Log 留痕。
3. 使用 break-glass 的阻塞必须记录票号、责任人、原因、撤销条件，且默认纳入后续治理任务。
4. 任何“先跳过检查”不能作为完成标准，只能作为有时限的临时措施。
